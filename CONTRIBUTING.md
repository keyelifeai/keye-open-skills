# 贡献指南

感谢你有兴趣为 `keye-claude-skills` 贡献技能！

---

## 🎯 贡献原则

### 核心原则
1. **实战验证** - 技能必须经过实际使用验证
2. **清晰描述** - 详细说明使用场景和效果
3. **分类准确** - 放入合适的分类目录
4. **格式规范** - 使用统一的 Markdown 格式

### 质量标准
- ✅ **有用** - 解决实际开发/创作问题
- ✅ **清晰** - 易于理解和使用
- ✅ **完整** - 包含必要的使用说明
- ✅ **有效** - 经过实际验证有效

---

## 📝 贡献流程

### 1. Fork 仓库
点击右上角的 Fork 按钮，创建你自己的副本。

### 2. 创建分支
```bash
git clone https://github.com/YOUR_USERNAME/keye-claude-skills.git
cd keye-claude-skills
git checkout -b feature/your-skill-name
```

### 3. 添加你的技能

#### 文件位置
根据技能类型，放入对应目录：
```
skills/
├── content-creation/     # 内容创作
├── development/          # 开发相关
├── ai-workflows/         # AI工作流
└── data-processing/      # 数据处理
```

#### 文件命名
使用小写字母和连字符：
- ✅ `article-generation.md`
- ✅ `code-review.md`
- ❌ `Article Generation.md`
- ❌ `code_review.md`

### 4. 技能文件格式

每个技能文件应包含以下部分：

```markdown
# 技能标题

## 技能描述
[1-2句话描述技能的用途和价值]

---

## 技能内容
```
[实际的技能内容]
```

---

## 使用场景
### 适用场景
- ✅ [场景1]
- ✅ [场景2]

### 特别适用
- [特殊情况说明]

---

## 使用方法
### 步骤 1: [标题]
[说明]

### 步骤 2: [标题]
[说明]

### 步骤 3: [标题]
[说明]

---

## 示例
### 输入
```
[示例输入]
```

### 预期输出
```markdown
[预期输出]
```

---

## 优化建议
[如何根据输出质量调整技能]

---

## 变体版本
### [变体1标题]
```
[变体技能内容]
```

---

## 常见问题
**Q: [问题]?**
A: [回答]

---

## 质量评估
### 优秀输出特征
- ✅ [特征1]
- ✅ [特征2]

### 需要改进的特征
- ❌ [特征1]
- ❌ [特征2]

---

## 更新记录
- **[日期]**: [更新内容]

---

**技能贡献者**: [Your Name](https://github.com/YOUR_USERNAME)
**质量评分**: X.X/10 (基于实际使用效果)
```

### 5. 提交更改
```bash
git add .
git commit -m "Add: [技能类型] - [技能标题]"
git push origin feature/your-skill-name
```

### 6. 创建 Pull Request
1. 访问你的 GitHub 仓库页面
2. 点击 "Compare & pull request"
3. 填写 PR 信息：
   - **标题**：`Add: [技能类型] - [技能标题]`
   - **描述**：说明这个技能的用途和效果
4. 提交 PR

---

## ❓ 常见问题

**Q: 技能需要多复杂？**
A: 复杂度适中，太简单缺乏价值，太复杂难以使用。

**Q: 可以提交多个相关技能吗？**
A: 可以，但建议每个技能独立成文件。

**Q: 如何判断技能质量？**
A: 看实际效果，是否稳定、准确、有价值。

**Q: 技能会被修改吗？**
A: 可能，为了保持质量和一致性，维护者可能会调整格式或优化内容。
